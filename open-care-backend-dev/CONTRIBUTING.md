# Contributing

Thanks for your interest in contributing to OpenCare Backend.

## Quick Start
- Fork the repository.
- Create a feature branch: `git checkout -b feature/your-feature`.
- Make your changes with clear, focused commits.
- Run tests: `mvn test`.
- Open a pull request with a clear description and screenshots/logs when relevant.

## Guidelines
- Follow existing code style and conventions.
- Add or update tests when behavior changes.
- Keep documentation in sync with code changes.
- Avoid large, unrelated refactors in the same PR.

## Reporting Issues
- Use GitHub Issues for bugs and feature requests.
- Provide steps to reproduce, expected behavior, and logs when possible.
